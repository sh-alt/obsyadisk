# ObsYaDisk — Синхронизация Obsidian ↔ Яндекс.Диск

Плагин для [Obsidian](https://obsidian.md), который синхронизирует ваше хранилище (vault) с [Яндекс.Диском](https://disk.yandex.ru) через REST API. Поддерживает git-based версионирование и разрешение конфликтов.

## Возможности

- **Двусторонняя синхронизация** — загрузка и скачивание файлов с отслеживанием изменений
- **Per-vault папки** — каждому хранилищу назначается своя папка на Яндекс.Диске
- **Git-версионирование** — автоматические коммиты при каждой синхронизации через isomorphic-git
- **Разрешение конфликтов** — три стратегии: спрашивать, приоритет локальной или удалённой версии
- **Автосинхронизация** — настраиваемый интервал (или только вручную)
- **История версий** — просмотр и восстановление предыдущих версий файлов
- **Контекстное меню** — история версий для конкретного файла из меню

## Установка

### Шаг 1 — Скопируйте файлы плагина

Создайте папку `.obsidian/plugins/obsyadisk/` внутри вашего vault и скопируйте туда три файла:

```
<ваш vault>/
└── .obsidian/
    └── plugins/
        └── obsyadisk/
            ├── main.js
            ├── manifest.json
            └── styles.css
```

> На Windows папка `.obsidian` может быть скрыта. Включите отображение скрытых папок в Проводнике.

### Шаг 2 — Включите плагин в Obsidian

1. Откройте **Settings** (⚙️) → **Community plugins**
2. Если показывается предупреждение "Safe mode" — нажмите **Turn on community plugins**
3. В разделе **Installed plugins** найдите **ObsYaDisk** и включите переключатель

### Шаг 3 — Зарегистрируйте OAuth-приложение на Яндексе

> Это нужно сделать один раз. Занимает ~3 минуты.

1. Откройте [oauth.yandex.ru/client/new](https://oauth.yandex.ru/client/new)
2. **Название:** любое, например `ObsYaDisk`
3. **Платформы:** выберите **Веб-сервисы** → в поле **Callback URI** вставьте:
   ```
   obsidian://obsyadisk-auth
   ```
   > ⚠️ Без этого адреса авторизация не вернётся в Obsidian.
4. **Доступы → Яндекс.Диск:** отметьте галками:
   - `cloud_api:disk.read`
   - `cloud_api:disk.write`
   - `cloud_api:disk.app_folder`
5. Нажмите **Создать приложение**
6. Скопируйте **ClientID** (и **Client secret**, если отображается)

### Шаг 4 — Авторизуйтесь в плагине

1. В Obsidian откройте **Settings → ObsYaDisk**
2. Вставьте **ClientID** в поле "Client ID приложения"
3. Вставьте **Client secret** в поле "Client Secret приложения" (если получили на шаге 3)
4. Нажмите кнопку **Авторизоваться** — откроется браузер
5. Войдите в Яндекс и разрешите доступ
6. Браузер автоматически вернёт токен в Obsidian — статус изменится на ✓ Авторизован

### Если автоматический возврат токена не сработал

Откройте раздел **"Ручной ввод токена"** в настройках плагина и вставьте токен вручную.
Получить токен можно по ссылке:
```
https://oauth.yandex.ru/authorize?response_type=token&client_id=<ВАШ_CLIENT_ID>
```
После авторизации Яндекс покажет токен в адресной строке (`#access_token=...`) — скопируйте его.

## Настройки

| Параметр | Описание |
|---|---|
| OAuth-токен | Токен для доступа к Яндекс.Диску |
| Папка на Яндекс.Диске | Путь к папке для этого vault (напр. `/ObsidianSync/Work`) |
| Интервал синхронизации | В минутах, 0 = только вручную |
| Стратегия конфликтов | `ask` / `prefer-local` / `prefer-remote` |
| Исключения | Glob-паттерны для пропуска файлов |
| Git-версионирование | Вкл/выкл автокоммиты |

## Команды

- **Синхронизировать с Яндекс.Диском** — запуск цикла синхронизации
- **Принудительная загрузка на Яндекс.Диск** — перезаписать всё на удалённом ⚠️ удалит файлы на Яндекс.Диске, которых нет локально
- **Принудительная загрузка с Яндекс.Диска** — перезаписать всё локально ⚠️ локальные изменения будут перезаписаны без возможности отмены
- **История версий (Git)** — просмотр лога коммитов
- **История версий текущего файла** — восстановление конкретного файла

## Разработка

```bash
npm install
npm run build     # production build
npm run dev       # dev build с sourcemaps
```

## Архитектура

- `src/main.ts` — точка входа плагина, команды, таймеры
- `src/yandex-disk-client.ts` — REST API клиент Яндекс.Диска
- `src/sync-engine.ts` — движок синхронизации с отслеживанием состояний
- `src/git-versioning.ts` — git-версионирование через isomorphic-git
- `src/conflict-modal.ts` — UI для разрешения конфликтов
- `src/version-history-modal.ts` — UI для просмотра и восстановления версий
- `src/settings-tab.ts` — вкладка настроек плагина
- `src/types.ts` — типы и настройки по умолчанию
- `src/utils.ts` — утилиты (хеширование, glob-matching, debounce)
